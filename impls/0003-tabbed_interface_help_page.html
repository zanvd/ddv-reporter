1. Overview

Wrap the existing single-page app in a two-tab ARIA tab pattern. The page <h1> stays above the tab bar. Tab "Poročilo" holds all of today's content (rewritten .app-intro, plus the unchanged Glava/KIR/KPR sections and download section); tab "Pomoč" holds static help — an app description, an open-source note linking to the repository, and one titled help article, "Pomnenje". Switching tabs is a pure show/hide of already-mounted DOM: the Poročilo form is never re-mounted, re-rendered, or re-validated, so every bit of in-progress state survives (spec §4). The only interactive code added is a small tab controller (click + arrow-key activation, roving tabindex, panel toggling). Everything else is static markup and CSS. Guiding principles: KISS/YAGNI — static copy stays static; no new abstraction, no persistence, no URL state.

4. tabs.js — behavior

Faithful to the approved mockup's controller. Pure DOM; no state module, no storage, no URL.

  • Signature: initTabs(tablistEl). Collects tablistEl.querySelectorAll('[role="tab"]'); each tab's target panel is document.getElementById(tab.getAttribute('aria-controls')).
  • selectTab(tab): for every tab, set aria-selected "true"/"false", roving tabIndex 0/-1, and its panel's hidden to !selected; then tab.focus().
  • Events: click → selectTab; keydown ArrowRight/ArrowLeft → preventDefault + select the next/previous tab with wrap-around. Native <button> supplies Enter/Space activation, satisfying the keyboard-operability requirement (§4) without extra handlers.
  • No deep-linking: writes no hash/query and reads none; the initial selected tab is whatever the static markup declares (Poročilo). Every load/reload starts on Poročilo (spec §4, out-of-scope §8).

5. State preservation — guaranteed structurally

The core requirement (spec §4, edge case §7) is met by construction, not by added code. main.js still calls renderGeneral, renderKirList, renderKprList, and renderPersistControls exactly once at boot, into the same containers — which now merely live inside #panel-porocilo. Auto-restore of the saved Glava record and the restored-fields revealErrors() call (spec 0002) run once at boot, before any tab interaction, and are untouched. Tab switching only flips the hidden attribute; there is no code path that re-mounts, re-renders, or re-validates the form on tab change. Therefore entered Glava/KIR/KPR values, added ledger rows, the revealed-vs-quiet error state, and the download message all survive a round-trip to "Pomoč" and back with zero extra work.

Why hidden (not display:none in JS or DOM removal). Toggling the native hidden attribute keeps the inactive panel's DOM and all listeners alive (state-preserving) and removes it from the accessibility tree and tab order (so hidden-panel controls aren't focusable). A single CSS rule .tabpanel[hidden]{display:none} backs it up for older UA safety.

7. Testing

  • No new node --test unit file. tabs.js is DOM-bound; the project already covers view/DOM behavior via the headless-Chrome smoke test rather than node unit tests (e.g. general.js/kirList.js have none). Adding a DOM shim just for tabs would violate KISS.
  • Existing unit suite stays green unchanged — no logic module is touched. Run: cd src && npm test.
  • Extend .claude/skills/run/smoke.mjs to assert:
    1. On load, #panel-porocilo is visible and #panel-pomoc is hidden; tab-porocilo has aria-selected="true".
    2. Clicking "Pomoč" shows the help panel (oss-note + "Pomnenje" article present) and hides the form; aria-selected flips.
    3. Arrow-key navigation moves selection between the two tabs (keyboard operability, spec §4).
    4. State preservation: type a Glava value, switch to "Pomoč" and back, assert the value is still present (and, optionally, that a revealed error stays revealed).

8. Build sequence

  1. index.html — insert tablist, wrap current content in #panel-porocilo, rewrite .app-intro, add static #panel-pomoc (verbatim copy). App still works (panel visible; Pomoč static) even before tabs.js exists.
  2. styles.css — add the tab-bar and Pomoč rules (§6).
  3. view/tabs.js — the controller (§4).
  4. main.js — import and call initTabs(...) after mounting.
  5. smoke.mjs — add the tab/state-preservation checks (§7); run unit tests + smoke test.

9. Assumptions & out-of-scope

  • Out of scope (spec §3/§8): deep-linking / URL-hash tab state, any help article beyond "Pomnenje", help search / table of contents / accordion, more than two tabs, dark theme, and any change to persistence, FURS fields, validation, serialization, or the DDV_KIR_KPR output.
  • Assumptions: (a) the repository URL is https://github.com/zanvd/ddv-reporter (spec §6.2); (b) the tab bar's accessible name is "Zavihki" (from the approved mockup; a non-user-facing ARIA label, not spec copy — flagged here in case a different label is preferred); (c) the section card styling (section{background:var(--bg-soft)…}) still applies to the three Poročilo sections inside the panel and needs no change, and the Pomoč panel uses <article> (not <section>) so that card style does not bleed into help content.
